    CSS:
    ​ 实践中优先外部样式表、平时练习可以使用内部样式表
    ​ 语法组成：1.选择器：基础：（元素、id、类、通配）
    ​ 复合：交集、并集
    ​ 关系：子元素、后代、兄弟
    ​ 属性选择
    ​ 伪类：特殊状态（：first-child）
    ​ 伪元素:特殊位置（::first-letter）
    ​ 权重问题：越具体权重越高
    ​ 2.声明块：样式继承问题
    ​ 页面布局：水平方向的布局：盒子的可视框部分包括：border、padding、content，三者影响可视框的大小；而margin虽然不影响可视部分，但是会占用实际空间；四部分相加=父元素的content。
    ​ 垂直方向的布局：不对高度有具体要求的情况下不需要定义高度：会被文本内容撑开，随着文本内容的增大而增大；外边距重叠问题
    ​ 行内元素不设置宽高，且其盒模型参数的修改不影响页面的垂直布局。
    ​ 链接之间的间距：1.不定义块元素高度，使用margin控制间距。
    ​ 2.定义块元素高度，控制文本大小自然产生间距。
    ​ 去除默认样式
    ​ 设置轮廓和圆角
    ​ 为了充分利用页面空间进行水平布局，引入浮动。但是浮动会随之带来水平塌陷的问题，可以通过开启bfc（overflow：hidden）或者设置clear(推荐方法)来解决。
    ​ 定位（position）:- 首先开启元素定位（position：），接着设置偏移量。

           - 相对定位：“灵魂出窍”，根据元素原始位置进行移动，依然占据原始空间，不脱离文档流（或者说是半脱离），但是会提高层级，所以会对其他位置上的元素进行覆盖。
                           - 绝对定位：根据包含块的位置进行移动，脱离文档流
           包含块( containing block )
           - 正常情况下：
               包含块就是离当前元素最近的祖先块元素
           - 绝对定位的包含块:
               包含块就是离它最近的开启了定位的祖先元素，如果所有的祖先元素都没有开启定位则根元素就是它的包含块
           - html（根元素、初始包含块）
                           - 固定定位：特殊的绝对定位；不同点在于固定定位根据视口进行移动。
           - 视口：指当前可视区域的左上角，与网页根元素不同；固定定位的元素不会随网页的滚动条滚动。
                           - 粘滞定位：与绝对定位类似，不同的是粘滞定位可以在元素到达某个位置时将其固定（固定定位）（兼容性一般，仅作为了解）
                               - 绝对定位元素的布局：在盒模型的基础上加上偏移量，使水平和垂直方面的布局都需要满足相应的等式。
                               - 开启了元素定位的模块可以使用z-index来定义模块的层级。
                               宏观上使用浮动，具体微调的时候使用position

    字体：font-family指定所用的网页字体
    -图标字体：font awesome(3种):1.通过伪元素来设置图标字体
    2.通过内联直接设置图标字体
    3.通过实体来使用图标字体 &#xf0f3;
    阿里的字体库：&#xe61c;
    行高：不设置的时候默认为1.33，行高指文字实际占有的高度，上下空隙均分，可以用此来使单行文字在元素中垂直居中（设置行高和高度一样）
    同时还可以用行高来指定多行之间的行间距（行间距 = 行高 - 字体大小）
    字体属性简写；font:bold(加粗) italic(斜体) 50px(大小)/2(行高) 微软雅黑, ‘Times New Roman’, Times, serif(字体族);
    其中某项省略不写的时候不是不设置，而是采用默认值；故若前面定义了其他值，则会被默认值覆盖并不生效。
    文本样式：text-align（水平对齐）
    vertical-align（垂直对齐）
    网页中引入图片时默认baseline对齐，会与父元素下边框存在一定距离，为此可以设置成其他垂直对齐方式。
    text-decoration(设置文本修饰)
    white-space (设置网页如何处理空白)
    text-overflow: ellipsis;(设置文本溢出处理方式)
    背景:background-color(背景颜色)
    background-image(背景图片)
    background-repeat(背景重复方式)
    background-position(背景位置)
    background-size(背景图片的大小)
    background-origin(背景图片的偏移量计算的原点)
    background-clip(背景的范围)
    background-attachment(背景图片是否跟随元素移动)
    简写要求: background-size必须写在background-position的后边，并且使用/隔开
    background-position/background-size
    background-origin background-clip 两个样式 ，orgin要在clip的前边
    线性渐变，颜色沿着一条直线发生变化
    background-image: linear-gradient(red,yellow,#bfa,orange);
    径向渐变：radial-gradient(大小 at 位置, 颜色 位置 ,颜色 位置 ,颜色 位置)

    过渡：transition-property: 指定要执行过渡的属性（有效值->有效值）
    transition-duration: 指定过渡效果的持续时间
    transition-timing-function: 过渡的时序函数
    transition-delay: 过渡效果的延迟
    简写：如果包含延迟，则两个时间中的后一个指代延迟
    动画: 首先设置好关键帧： @keyframes name {
    from{} 动画的起始位置
    to{} 动画的结束位置
    20%{} 关键帧的百分比
    }
    设置动画:animation-name 对当前元素生效的关键帧的名字
    animation-duration 动画的执行时间
    animation-delay 动画的延时
    animation-iteration-count 动画执行的次数
    animation-direction 动画运行的方向
    animation-play-state 动画的执行状态
    animation-fill-mode 动画的填充模式
    变形:变形不会影响到页面的布局
    X、Y轴的平移：transform:translateX(100%);
    Z轴的平移： html{

    transform: translateZ(800px);
    旋转：transform: rotateY(180deg);
    同样要想有3d效果需要先设置视距

    scaleY() 垂直方向缩放

    5.常用的一些多功能函数

FLEX: 是CSS中的又一种布局手段，它主要用来代替浮动来完成页面的布局。flex可以使元素具有弹性，让元素可以跟随页面的大小的改变而改变。
- 设置弹性容器：display:flex，其包含的元素自动变为弹性元素。
- flex-direction：指定容器中弹性元素的排列方式，引申出主轴和侧轴的概念。
- flex-grow、flex-shrink分别设置弹性元素的伸展（收缩）系数。
弹性容器的样式：display: flex 设置弹性容器
flex-wrap: wrap 设置弹性元素是否在弹性容器中自动换行
flex-direction: column 设置容器中弹性元素的排列方式
-flex-flow: wrap 和 direction 的简写属性
justify-content：如何分配主轴上的空白空间（主轴上的元素如何排列）
align-content:辅轴空白空间的分布
align-items: 元素在辅轴上如何对齐
- align-self: 用来覆盖当前弹性元素上的align-items
弹性元素的样式：flex-basis: auto 设置元素在主轴上的基础长度
flex-grow: 1; 设置元素的伸展系数
flex-shrink: 1; 设置元素的缩减系数
简写方式：flex 增长 缩减 基础;
- initial：”flex: 0 1 auto”.
- auto ： “flex: 1 1 auto”
- none ： “flex: 0 0 auto”
像素：- 在前端开发中像素要分成两种情况讨论：CSS像素 和 物理像素
- 默认情况下：
视口宽度 1920px（CSS像素）
1920px（物理像素）
- 此时，css像素和物理像素的比是 1:1

        - 放大两倍的情况：
            视口宽度 960px（CSS像素）
                    1920px（物理像素）
                    - 此时，css像素和物理像素的比是1:2

        - 我们可以通过改变视口的大小，来改变CSS像素和物理像素的比值
        - 分辨率、窗口大小、像素点的大小
        - 默认情况下，移动端的网页都会将视口设置为980像素（css像素）
        - 在移动端开发时，不同的屏幕，像素点的大小是不同的，像素越小屏幕会越清晰
        - 希望在不同的设备中看到的图像大小差不多：
            - 将网页的视口设置为完美视口：
                <meta name="viewport" content="width=device-width, initial-scale=1.0">。
        - 希望在不同的设备中看到的显示效果相同：
            -更改前端开发的单位设置：
                将px改为vw（视口宽度）
                html{
                    font-size: 5.3333vw;
                }
                .box1{
                    width: 18.75rem;
                    height: 0.875rem;
                    background-color: #bfa;
                }
响应式布局： - 网页可以根据不通的设备或窗口大小呈现出不同的效果
            - 使用响应式布局，可以使一个网页适用于所有设备
            关键：媒体查询（常用的断点）
                语法： @media 查询规则{}    
                @media only screen and (min-width: 500px) and (max-width:700px){
                    body{
                        background-color: #bfa;
                    }
                }
